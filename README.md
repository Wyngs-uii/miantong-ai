# 面通 AI

面向低经验大学生的 AI 简历生成与岗位匹配工具。用户输入求职目标、岗位 JD 和少量真实经历后，系统会通过引导式追问补充信息，生成可编辑、可导出的岗位定制简历。

线上体验：<https://shili-ai-resume.wuy228511.chatgpt.site/>

## 产品目标

面通 AI 主要解决三类问题：

- 缺少正式实习经历，不知道课程、项目和校园活动中哪些内容可以写进简历。
- 经历描述口语化、重复或缺少结果，难以形成专业且严谨的简历要点。
- 同一份简历直接投递不同岗位，无法体现与目标 JD 的匹配关系。

系统坚持以用户提供和确认的事实为基础，围绕经历提炼、岗位匹配、内容优化和简历导出形成完整流程。

## 已实现功能

- 分步采集求职目标、基础信息和多段经历。
- 支持项目、实习、校园及竞赛等经历类型。
- AI 结构化提炼、动态追问和单段经历持续完善。
- 已有内容累积保存，避免后续生成覆盖此前补充的信息。
- JD 关键词覆盖、缺失项和修改意见实时同步。
- 语义相近要点自动去重，并减少重复动作动词与句式。
- A4 ATS 单栏简历实时预览。
- PDF、JPG 和 Word 导出。
- 本地草稿保存和多模型设置。
- DeepSeek、OpenAI、Gemini 接口切换与连接检测。
- 本地证件照处理，支持纯白与纯蓝背景。
- 桌面端和移动端响应式布局。

## 核心流程

```text
求职目标
  → 基础信息
  → 多段经历采集
  → AI 提炼与补充追问
  → JD 修改意见
  → 简历生成与编辑
  → PDF / JPG / Word 导出
```

## 技术栈

- Next.js App Router、React 19、TypeScript
- vinext、Cloudflare Workers、Sites

## 本地运行

环境要求：Node.js `>=22.13.0`。

```bash
npm install
npm run dev
```

生产构建与测试：

```bash
npm run lint
npm test
```

模型 API Key 可以在网站的“模型设置”中录入、切换和检测。请勿将真实 Key 写入代码或提交到 Git。

## 数据与隐私

- 当前简历草稿主要保存在用户浏览器本地。
- 模型 Key 通过安全 Cookie 传递，不在前端页面回显完整内容。
- 证件照抠图在当前设备完成，原始照片不会作为项目文件保存。
- 生产使用时仍应根据部署环境补充隐私政策、数据删除说明和访问控制。
