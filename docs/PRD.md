# 面通 AI 产品需求文档（PRD）

- 文档版本：V2.0
- 更新日期：2026-07-15
- 产品阶段：一期内测完成，进入二期质量与长期使用能力建设
- 适用端：响应式 Web，后续可复用至小程序

## 1. 产品定义

面通 AI 是面向低经验大学生的简历生成与岗位匹配工具。用户输入目标岗位、JD 和少量真实经历后，系统通过引导追问、事实提取、语义匹配和专业改写，生成可编辑、可导出的岗位定制简历。

产品核心承诺：

1. 降低经历整理和简历写作门槛。
2. 将课程、项目、校园活动和短期实践转换为专业表达。
3. 基于 JD 调整表达重点，而不是机械堆叠关键词。
4. AI 只提炼已确认事实，不凭空添加职责、技术、数据或成果。
5. 用户已经确认或手工修改的内容不会被后续生成覆盖。

## 2. 目标用户与使用场景

### 2.1 目标用户

- 缺少正式实习经历的在校大学生。
- 有课程、竞赛、校园或个人项目，但不会写成简历的求职者。
- 需要针对不同岗位快速调整简历内容的校招用户。

### 2.2 核心场景

1. 首次从零生成一份可投递简历。
2. 根据目标 JD 挖掘现有经历中的相关证据。
3. 对单段经历持续补充，不丢失已有内容。
4. 为不同岗位复用经历并生成不同版本。
5. 导出 A4 PDF、JPG 或可编辑 Word。

## 3. 产品目标与非目标

### 3.1 二期目标

- 经历和分析结果可稳定保存、重复调用与版本恢复。
- JD 匹配由字面匹配升级为可解释的概念匹配。
- 生成内容具备事实依据、表达差异和语义去重能力。
- 同一经历可连续完善，任何操作都不因省略字段而删除旧信息。
- 形成可验收的导出、移动端和无障碍质量标准。

### 3.2 暂不纳入

- 自动投递、招聘账号托管和代用户沟通。
- 为提高分数而生成不存在的经历或数据。
- 企业招聘管理、人才库和复杂团队权限。
- 模型训练或自建大模型。

## 4. 核心流程

1. **求职目标**：填写目标岗位、公司和 JD。
2. **基础信息**：填写个人、教育、联系方式和证件照。
3. **经历采集**：新增多段经历，录入原始描述和时间范围。
4. **修改意见**：分析 JD 概念、已覆盖证据、缺口和建议补充项。
5. **生成简历**：编辑要点、实时预览、保存和导出。

用户可从第 5 步返回任一经历继续完善；返回和刷新不得造成正文、事实、追问、匹配证据或手工编辑丢失。

## 5. 功能需求

### FR-01 求职目标与 JD 解析

- 支持目标岗位、公司和完整 JD 输入。
- 将 JD 转换为稳定的“能力概念”，每个概念包含名称、别名、重要度和原文证据。
- 区分核心职责、工具技能、业务场景、软能力和加分项。
- JD 未变化时，评分概念集合和权重不得因重新生成而改变。

验收标准：

- “收集学生意见并分类整理”可匹配“用户反馈”。
- “Excel 统计问题分布”可按置信度匹配“数据分析/数据统计”。
- JD 未出现且无可靠证据的“复盘”等词不得被判为已覆盖。
- 每个覆盖项都能展示来自 JD 和经历的证据片段。

### FR-02 基础信息与联系方式

- 支持姓名、岗位、手机、邮箱、微信、到岗时间、毕业时间、教育信息。
- 手机号允许空格、短横线和国家区号，预览及导出必须完整显示。
- 输入、预览、PDF、JPG、Word 使用同一份标准化数据。

验收标准：输入 `138 0000 0000` 后，四种呈现位置均可见且内容一致。

### FR-03 证件照工作室

- 支持上传日常照片、人物抠图、裁切和白底/蓝底切换。
- 背景必须使用纯色填充：白色 `#FFFFFF`，蓝色采用产品统一色值。
- 原图和处理图默认仅保存在本机草稿；上传前明确告知处理位置。
- 提供重做、恢复原图和删除照片。

验收标准：以固定测试照片导出后，四角和非人物区域像素颜色一致，边缘无原背景残留块。

### FR-04 多段经历资产

- 支持项目、实习、校园、竞赛、课程实践等类型。
- 每段经历拥有稳定 ID，新增、切换、删除和排序互不影响。
- 保存标题、组织、时间、原始描述、确认事实、补充回答、角色建议、简历要点和分析版本。
- 刷新后恢复上次编辑的经历和步骤。

验收标准：连续建立 5 段经历并分别完善，任意切换和刷新后内容不串段、不丢失。

### FR-05 AI 角色/职责联想

- 用户可以手工输入，也可点击有依据的 AI 推荐词。
- 每个推荐词必须绑定事实证据和置信度。
- 无证据时只提出追问，不直接写入“负责”“主导”“验证”等职责。
- 点击推荐词后仍可删除或修改。

验收标准：若用户只描述访谈和整理反馈，系统可推荐“用户访谈、反馈整理”，不得无依据推荐“功能验证”。

### FR-06 引导补充与非破坏式合并

- 根据缺失字段生成最多 4 个短问题。
- 用户可不回答追问，直接修改要点并保存。
- 补充生成只允许新增事实或在用户明确要求时更新指定要点。
- 模型省略旧字段不得触发删除；既有事实、回答和手工要点必须按 ID 合并。
- 相同补充回答不得被重复追加。

验收标准：手工修改要点后补充一次、重复补充一次并刷新，手工要点仍存在，事实和补充记录无重复。

### FR-07 简历要点生成与去重

- 每条要点应包含行动、方法或工具、结果中的至少两个要素。
- 避免“负责、参与、完成、整理、推进”等动词连续重复。
- 禁止用空泛句补数量，如“围绕岗位需求推进核心任务”。
- 同时进行字符级、关键词级和语义级去重。
- 相似度过高时合并信息更完整的一条，而不是简单追加。

验收标准：同一经历连续生成 3 次，要点数量不因同义改写无限增长，重复含义只保留信息最完整版本。

### FR-08 JD 匹配与稳定评分

评分由确定性规则计算，大模型只负责提取概念与证据，不直接输出最终分数。

建议权重：

- 核心职责概念覆盖：40%。
- 工具与技能覆盖：25%。
- 经历证据完整度：20%。
- 表达质量与可验证结果：15%。

匹配等级：

- 原词命中：1.00。
- 受控别名命中：0.90。
- 行为证据规则命中：0.75。
- 模型语义判断：置信度不低于 0.80，且必须返回证据。

稳定性规则：

- JD 不变时分母固定。
- 添加相关且真实的经历证据后，单项覆盖不得下降。
- 文案重写不应改变已有事实的覆盖状态。
- 任何分数变化都应展示新增、失去或权重变化的原因。

### FR-09 修改意见

- 展示匹配总分、已分析经历数、已覆盖概念、待补概念和证据。
- 缺口建议应转化为可回答的问题，而不是要求机械复制 JD 关键词。
- 用户可从此页新增实习/项目经历，或返回指定经历补充。
- 修改完成后数据自动同步并重新计算。

### FR-10 简历编辑器与预览

- 要点均可直接编辑、删除、排序和重新生成。
- “保存并继续”不依赖追问是否填写。
- 预览与导出共用排版数据，A4 单栏 ATS 模板为默认。
- 手机端为单栏编辑流，不出现横向滚动；预览置于表单之后。

### FR-11 导出

- PDF：A4 纸张、内容可搜索、分页可控。
- JPG：按 A4 比例导出，宽高和边距与 PDF 一致，照片不失真。
- Word：A4、可编辑、照片尺寸受控，版式与 PDF 尽量一致。
- 导出前提供页数、溢出和空白页检查。

验收标准：用短、中、长三套固定简历数据，在 Word、预览和 PDF 中逐页对比；无巨幅照片、内容缩放异常、截断或多余空白页。

### FR-12 模型设置

- 支持 DeepSeek、OpenAI、Gemini 的选择、Key 保存、检测、替换和清除。
- 页面和接口状态不得返回完整 Key；最多显示掩码后四位。
- 调用失败区分 Key 无效、余额/限流、网络和返回格式错误。
- 任一模型不可用时提供可理解的回退提示。

### FR-13 草稿、分析与版本管理

- 草稿正文和分析结果必须在初始化完成后再写回存储，防止空状态覆盖旧数据。
- 数据结构包含 schemaVersion，并支持迁移。
- 自动保存显示时间；提供恢复上版和清空本机数据。
- 二期后半段可增加登录与云端多版本，但不得阻塞本地使用。

## 7. 非功能要求

### 安全与隐私

- 仓库、日志、接口和前端状态不得包含明文 API Key。
- 照片和简历属于敏感个人信息，默认本机保存，并提供一键清除。
- 对导出文件名、XML 和模型输出进行转义，防止注入。

### 可访问性

- 所有输入框具备可读标签，错误信息与字段关联。
- 键盘可完成五步流程、模态框和导出菜单操作。
- 文本和按钮对比度满足 WCAG AA。

### 性能与可靠性

- 首屏可操作时间目标不超过 3 秒。
- 本地编辑反馈不超过 100 毫秒。
- 模型调用超时、重试和取消均有状态提示。
- 保存操作幂等，重复点击不产生重复事实或要点。

## 8. 数据指标

- 首份简历完成率。
- 单段经历平均补充次数与完成时间。
- AI 要点保留率、用户修改率和删除率。
- 匹配度提升的中位数及其证据来源。
- 导出成功率及 PDF/JPG/Word 分布。
- 刷新后恢复成功率和数据丢失率。
- 模型错误率、JSON 校验失败率和回退率。

不得将“分数越高”单独作为成功指标，应同时观察用户保留率和事实证据完整度。

## 9. 测试结果转化的优先级

### P0：公开推广前必须完成

1. 修复分析结果刷新丢失。
2. 修复手机号格式识别与预览缺失。
3. 建立固定 JD 概念分母和可解释语义匹配。
4. 禁止无证据的角色/职责推荐。
5. 完成非破坏式合并与补充幂等去重。

### P1：二期首个迭代完成

1. 语义要点去重和空泛表达拦截。
2. 分数单调性与变化原因展示。
3. 固定样例的 PDF/JPG/Word 视觉回归。
4. 固定照片的白底/蓝底像素回归。
5. 完成表单标签和键盘操作优化。

### P2：长期使用能力

1. 经历资产库与一岗一简历。
2. 登录、云端保存和历史版本。
3. 小程序适配与跨端同步。

## 10. 发布验收门槛

- 自动化测试、生产构建、密钥扫描全部通过。
- P0 缺陷为 0，P1 无阻塞主流程的问题。
- 5 段经历连续编辑、刷新和重复补充无数据丢失或重复。
- 20 组 JD/经历基准样例的语义匹配经人工复核，错误覆盖和漏覆盖均有记录。
- 三种导出格式分别用至少 3 套数据打开验收。
- 手机、平板、桌面三档宽度无横向溢出。
